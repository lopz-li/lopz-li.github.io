LGPL协议详解（转载）

发表于 2025-08-10 分类于 记录

看到一篇关于LGPL协议的简洁介绍，转载记录一下。

原文：https://www.devbean.net/2011/12/lgpl-explain/

Qt有两个许可证：LGPL和商业协议。这两个协议在现在的Qt版本中的代码是完全一致的（潜在含义是，Qt的早期版本，商业版的Qt通常包含有一些开源版本所没有的库，比如QtSingleApplication这个库）。所以现在对于普通开发人员和部分商业公司来说，使用LGPL版本的Qt可以节省很大的开销。这两个版本最大的区别在于，前者是免费的，后者是收费的。既然代码都是一致的，所以费用就要是用来购买Qt的售后服务和培训等等相关服务。

现在我们是来说一下版权的问题。LGPL是一个开源协议，因此，有人会担心LGPL能否用于开发闭源程序，能够拿来卖钱。尽管现在国内有些公司不是很重视这方面的问题，不过，如果你违反了协议，某一天被别人发来一纸律师函的时候，真的是欲哭无泪了哦。所以，我们还是先来研究一下这个协议，LGPL究竟能不能用于开发闭源程序。

至于什么是LGPL协议，这里就不再多说了，我们关心的是，如果使用LGPL协议开发商业程序。请注意，这里所说的闭源程序，是指不以某种形式开放源代码，也就是说，用户（包括其他开发者）不能获取其源代码的程序。首先说明一点，LGPL协议是一个商业友好的协议。这里的含义是，你可以用LGPL协议开发商业程序，当然也可以是非商业的闭源程序。但是，它是有一些限制的。这就是我们要讨论的重点。

既然我们已经对其定性，那么我们直接进入主题：使用LGPL协议开发闭源程序，如果你使用动态链接的形式，那么，你可以以任何形式（商业的、非商业的、开源的、非开源的等等）发布你的应用程序。

如果你因某种原因必须静态链接一个基于LGPL协议发布的库（以下我们简称为LGPL库），那么，你有义务进行下面的工作：

1. 你必须在你的文档中说明，你的程序中使用了LGPL库，并且说明这个库是基于LGPL发布的
2. 你必须在你的应用程序发布中包含一份LGPL协议，通常就是那个文本文件
3. 你必须开放使用了LGPL库代码的所有代码，例如某些封装器。但是，其他使用这些封装器的代码就不需要开放了
4. 你必须包含你的应用程序的余下部分的目标文件（通常就是我们所说的.o等等），或者是其他等价的文件，源代码并不是必须的

是不是很难理解呢？我们来详细说明一下。

第一条很容易理解；第二条也很容易理解，你可以在这里找到LGPL协议的内容，复制下来随你的程序一起发布就可以了。第三条就不那么好理解了。简单来说，LGPL协议要求，如果你的类使用了LGPL库的代码，那么必须把 这个类开源。例如，如果你的程序app.exe每个源文件都使用了LGPL库的代码，那么你的所有源代码都要开源。为了避免这种情况，我们通常编写一个封装器，把LGPL库的代码封装起来，这样就只需要开放这个封装器的代码，而其他使用了这个封装器的代码就不需要开放。第四条是对第三条的一种补充：那些使用了封装 器的程序不需要开源，但是你必须把你编译的那些中间文件开放出来，比如gcc编译器的那些.o文件。

为什么LGPL协议要这样规定呢？原来，LGPL所做的工作是，它保证了库的使用者能够有这样一种能力：修改你使用LGPL库函数的方式（那些封装器就是你使用LGPL库的方式，现在已经开源了），重新编译这些代码，然后重新对程序进行连接（连接所需要的目标文件也是包含了的，这是第四条规定的），就可以得到一 个新的可执行程序。

STM8S单片机硬件I2C程序卡死的调查

发表于 2024-12-02 分类于 记录 ， 嵌入式 ， STM8

今天调试STM8S上的I2C通讯时观察到了一些现象，在此记录一下。

STM8单片机的硬件I2C有一些错误产生机制，例如主机向从机写数据时，如果收到NACK，应该会置位AF标志位并产生中断。但调试时发现如果总线上拉不正常时，I2C外设不会置位标志位和产生中断，推测应该是无上拉时没有信号边沿，无法触发状态转换，如果代码中有轮询标志位或使用中断驱动的，会导致程序死循环而不能正常运作。因此出现卡死现象时应先检查总线信号上拉是否正常。

给魔百盒CM311-1s YST安装armbian系统

发表于 2024-11-15 分类于 记录 ， armbian

之前买的二手刷机电视盒子太卡了，于是刷了个armbian当服务器用，在此记录一下过程。

盒子是中国移动魔百盒，型号CM311-1s YST，网上能找到很多CM311-1a YST的资源，但1a和1s的SoC型号不一样，不能完全照搬，需要找对应的正确固件。

研究了一下，确定刷机过程应为：线刷安卓固件 -> 烧录armbian系统U盘 -> 用开心电视助手控制盒子从U盘启动 -> 在U盘系统环境下安装armbian到EMMC

需要准备的材料：

  1. amlogic芯片USB刷机工具：可搜索Amlogic USB Burning Tool，下载一个较新的版本即可
  2. 双头USB-A数据线：可购买或自制（直连线两头线序相同）
  3. 镊子：金属导电镊子，用于短接进入刷机模式
  4. 安卓线刷固件：适用CM311-1s YST型号的安卓线刷包
  5. U盘：容量须大于解压后的armbian固件大小
  6. U盘烧录工具：可用rufus、balenaEtcher等
  7. armbian固件：可到https://github.com/ophub/amlogic-s9xxx-armbian下载需要的Release版本
  8. 开心电视助手：可搜索开心电视助手，下载较新的版本

部分固件及刷机工具网盘链接

下载好安卓线刷固件后解压得到后缀img的固件文件，打开Amlogic USB Burning Tool刷机工具，在文件菜单中导入img固件。

拆出盒子主板，双头USB-A线一头插到盒子上离网口较近的USB口，先用镊子短接板子背面4R12电阻，再将线的另一头插入电脑，刷机工具上应该能看到显示板子连接成功，此时即可拿开镊子。
（我的这个盒子插上数据线后就通电连接上了，不需要另外接电源，别的盒子可能要先通电之后再插数据线）

板子连接上之后即可开始刷新固件，刷新完成后会有提示，点停止后断开数据线，然后重启盒子进入安卓系统。

下载好armbian系统固件，通过U盘烧录工具（可用rufus、balenaEtcher等）烧录到U盘上。

进入安卓后可以在盒子网络设置界面或者在路由器设置界面上得到盒子的IP，打开开心电视助手，以默认端口5555连接盒子IP。

开心电视助手连接盒子成功后，在调试选项卡界面中的amlogic类别里选择从U盘/SD卡启动，等盒子重启后迅速插入系统U盘（此举是为了避免安卓系统修改U盘文件权限），让盒子从U盘启动armbian系统。

盒子从U盘启动armbian系统后，可以在路由器设置界面上看到armbian名字的IP，可用终端工具ssh登录，用户名root，密码1234，根据提示进行系统初始设置。

执行armbian-install命令将armbian系统安装到EMMC，设备序号选122，安装完成后重启盒子。

如果安装系统到EMMC后出现根目录下部分目录的用户/组显示1023，说明被安卓系统修改了U盘文件的权限，此时可以使用U盘烧录工具重新烧录一次armbian固件，然后插上U盘，盒子会自动从U盘启动，重新安装系统到EMMC即可。

注意：CM311-1s YST型号的盒子安装armbian之后HDMI是没有输出的，只能通过远程登录进行操作，IP可在路由器设置界面上获得。

NetworkManager管理WiFi但找不到无线网络的解决方法记录

发表于 2023-12-07 分类于 记录 ， Debian

最近重装Debian 12，配置NetworkManager的过程中遇到了无法找到无线网络的问题，在此记录一下解决过程。

全新安装Debian 12后，网络连接默认不由NetworkManager管理，需要手动修改/etc/NetworkManager目录下的配置文件NetworkManager.conf，将ifupdown段的managed修改为true，然后sudo systemctl restart NetworkManager重启服务。

重启服务后发现完全看不到任何WiFi网络，首先确认wpa_supplicant服务是否存在和启动：sudo systemctl status wpa_supplicant。确认服务正常后检查一下NetworkManager服务的状态：sudo systemctl status NetworkManager。此时发现报Couldn’t initialize supplicant interface错误，再检查wpa_supplicant服务状态，发现报ctrl_iface exists and seems to be in use，判断应该是已经通过iface配置了连接参数，导致状态异常。

鉴于以上分析，修改/etc/network/interfaces，注释掉无线网卡的iface配置行，保留接口声明行，然后分别重启networking、wpa_supplicant和NetworkManager服务，然后就能正常搜索和连接WiFi了。
