Debian根分区扩容

发表于 2022-11-25 更新于 2023-04-24 分类于 记录 ， Debian

因为感觉Debian分区的容量有些不太够用，日前清空了一个不常用的Windows分区，将空闲空间合并到Debian的根分区，在此记录一下过程。

调整根分区的风险很大，因此提前tar备份了重要的数据到仓库盘，也曾尝试用DiskGenius镜像的方式备份，失败了…

首先清空Windows分区并删除分区得到空闲空间，空闲空间位于Debian根分区之前…

因为DiskGenius镜像文件方式失败，所以放弃删除分区合并空间再镜像文件恢复的方式，试了Debian下的磁盘工具，因为需要先卸载分区才能调整，再次放弃…

尝试直接用DiskGenius调整分区大小，检测Debian根分区时报告发现文件系统错误，到Debian下检测并未发现问题，放弃此方式…

尝试先将空闲空间分区后，DiskGenius克隆现有Debian分区到新分区成功，发现此时新分区的卷UUID和原分区完全一样，重启后Debian还是挂载老分区为根目录。尝试在Grub启动系统时修改根文件系统指向，启动Debian成功，挂载新分区为根目录，简单尝试各项功能都正常。于是删除原Debian分区，再用DiskGenius调整新分区大小，合并全部空闲空间，重启用原Grub配置启动到新分区成功。

逐个确认各软件及功能是否正常，大部分正常运行，发现kvm虚拟机无法启动，报找不到操作系统，尝试从备份文件恢复虚拟机硬盘镜像文件，再次启动成功。

收获：事前备份很重要！！！ 事后确认很重要！！！

Windows 7下安装Visual Studio 2017过程

发表于 2022-11-24 更新于 2023-04-24 分类于 记录 ， Windows

Windows 7下安装Visual Studio 2017比想象的复杂，安装过程中会出现证书过期，装好后会出现登录账户白屏卡死等问题，因此在此记录一下安装过程。

系统要求

必须要最新的SP1版本Win7才可以安装

安装系统补丁

必须要安装kb4474419和kb4490628两个系统补丁，安装完后需要重启以生效。

补丁的作用应该是更新证书，避免后续安装过程出现证书过期等引起的失败。

安装.Net Framework

必须要安装Microsoft .NET Framework 4.6以上版本，可到微软官网下载安装。

下载安装程序

到visualstudio.com下载最新版本的Visual Studio 2017安装程序，企业版和专业版均可。

运行安装程序

右键菜单中以管理员身份运行下载好的Visual Studio 2017安装程序。

选择版本

根据所下载的安装程序的不同，安装过程启动后会直接进入企业版/专业版的安装包选择界面。此时可关闭安装包选择子窗口，在父窗口中选中《可用》标签去选择需要安装的版本（社区版/专业版/企业版），可启动对应版本的安装包选择界面。

登录账户

Visual Studio 2017需要登录账户更新许可证才能正常使用（否则只能试用一段时间），但登录时经常会出现白屏卡死等现象，猜测是由于登录服务器在国外的原因，在IE的Internet选项中去设置系统代理，然后多尝试几次即可登录成功，IE必须要更新到最新的11版本，可到微软网站上下载安装包。

特斯拉Model 3海南半环岛

发表于 2022-08-25 更新于 2023-04-24 分类于 记录 ， 特斯拉

八月初是老婆生日，所以7月末我们来到了海南，租了一辆2022款特斯拉Model 3准备东线自驾…

整个行程7天，计划是从海口开到三亚，一路开一路玩过去，不赶路，所以续航什么的显得不是很重要。行程期间海南天气很好，太阳非常大，2022款的热泵空调感觉很给力，车子开动的时候温度稍微调低一点都会觉得冷。就是停车充电的时候可能是散热压力比较大，空调的效果就不是那么好。

这次的路线有很多非高速的省道、乡道之类的路，需要手动驾驶，要长时间保持油门深度，时间长了以后感觉腰会有点疼，可能是右腿长时间用力的原因。

快返程时三亚出现了疫情，经过各种辗转折腾终于可以开车离开，到海口赶飞机，时间比较紧，所以全程处于赶路的状态。这里要提一下特斯拉的车机导航，出发的时候用的车机导航，但在海棠湾上高速的时候发现出三亚方向的匝道已经封闭了，跟着导航转了两个匝道口都一样。于是果断改用手机高德导航，跟着走了一段省道在下一个开放的路口上了高速，所以很多人还是要用手机导航也是有原因的。整个返程几乎全程高速，大约240公里路程，车是满电（表显439）出发的。快到海口的时候瞟了一眼，开了210公里，表显还剩200多一点，这感觉和之前租的20款明显就不一样了，心里一点都不慌，22款的区别还是挺大的。另外22款的加速是6.1秒，实际加速感受确实比20款的5.6秒要差一点，但超车的信心还是非常足的。

这次体验的时间算是比较长了，对特斯拉油门硬的印象也进一步加深了…

从Ubuntu迁移到Debian：恢复软件配置

发表于 2022-07-15 更新于 2023-04-24 分类于 记录 ， Debian

新系统装好之后需要将旧系统中的软件配置都恢复过来，好在Linux下软件的配置信息存储位置都很容易找到，多数直接复制即可。

ssh秘钥

git等软件为了方便使用经常会配置ssh公私钥，这样就可以不用频繁输入密码。ssh的秘钥一般都存储在用户目录下的.ssh目录中，直接将旧系统中的整个目录复制过来即可。

git

git配置信息一般存储在用户目录下的.gitconfig文件中，从旧系统中复制文件即可。

vim/gvim

vim/gvim配置信息一般存储在用户目录下的.vimrc/.gvimrc中，相关文件一般存储在用户目录下的.vim目录下，从旧系统中复制过来即可。

aegisub

aegisub配置信息存储在用户目录下.aegisub目录中的config.json文件中，从旧系统复制文件即可

wine软件

旧系统中使用wine运行的windows软件都安装在用户目录下.wine目录中，wine环境的配置信息也都存储于此，直接复制此目录应该可以完美移植wine环境，但我自己是重新安装了所有的wine软件。

PATH配置等

PATH环境变量等配置信息一般都存储在用户目录下的.bash_profile等文件中，需要手工从旧系统中将配置值复制过来，根据新系统路径状态修改适配。

初次体验特斯拉Model 3

发表于 2022-07-11 更新于 2023-04-24 分类于 记录 ， 特斯拉

上周租了两天特斯拉Model 3开了开，属于是第一次长时间体验了，记录一下。

车型

这次租的是2020款的最普通那款Model 3，携程下单，收费上门送取车。总体感觉服务不错，交还车都很顺利，车子也比较干净，车况很好。

驾驶

  • 加速

电车的加速响应确实很快，可以说是随心所欲的感觉，吊打各种自动挡。空旷的路上试了下加速能力，没踩到底，但感觉已经非常猛了。路上各种超车毫无压力。

  • 单踏板

特斯拉争论最多的就是所谓的单踏板模式，其实就是强能量回收加上Auto Hold。由于能量回收力度很强，并且跟加速踏板深度有关，速度低至临近停止时，车辆会自动控制刹车刹死。这样大部分情况下，驾驶者只需要控制加速踏板即可，“单踏板模式”由此得名。实际上两天开下来，感觉这种驾驶方式还是很容易适应的，松加速踏板减速的特性只需要几分钟就适应了，路上遇到比较紧急的情况时，右脚在加减速踏板之间的切换也都是下意识的，没有网上说的那么多问题。在山路下坡等一些情况下，能量回收的刹车力度也会有不足的现象，需要踩刹车进一步减速。全程下来觉得一个踏板搞定的这种方式一定程度上还是比普通油车要省劲的。

  • 辅助驾驶

高速和快速路上尝试了特斯拉标配的辅助驾驶AP功能。刚开始用的时候还是有些怕的，对它的能力边界没什么底，适应了一段时间之后就比较放心了。租车的两天时间内基本都是晴天，路面视野非常好，特斯拉能很好地识别车道线并保持在车道中间。遇到前方有车的时候，能自动减速，此时可先打转向灯，然后手动控制超车，车道保持会失效，但速度保持扔继续工作，后方无车干扰时，车会自动加速，驾驶者只需要控制好方向即可。超车完成后，需要再次手动启动车道保持功能。

开启辅助驾驶时如果觉得车速较慢，可以踩加速踏板加速，这种情况下辅助驾驶系统遇到前方障碍时不会自动刹车，实际体验时，这种情况下能量回收模式貌似和不开辅助驾驶时不太一样，加速踏板的脚感和正常驾驶时也有所不同，需要十分注意。

车身不在车道正中时如果开启辅助驾驶，在靠边的车道上会有一股很大的力量将方向盘拉到撞向车道外边线的方向，让人觉得车子要甩出车道，但车身在车道正中的时候开启则不会有这种现象，需要十分注意。

开启辅助驾驶一段时间之后，由于注意力逐渐转移，亲身体验驾驶者的安全意识会有一定程度的松懈，看手机之类的小动作持续时间会显著延长，这一点需要非常注意！！！

充电

此次租车中间一共充了四次电，分别是一次景区的慢充、两次高速服务区快充、一次商业体停车场快充，总体感受还是不错的。景区停车场提供了很多慢充桩，功率7kW，绝大部分可用，进去玩时插上，出来时可以充不少里程，挺方便的。高速服务区貌似都是电网公司的快充桩，会有部分桩不支持特斯拉、小鹏等一些车型，充电速度一般，峰值50多kW，但大部分时候只有30多kW。商业体停车场的是开迈斯快充，最高到90多kW，速度很快。

其他

空调效果不错，打开之后很快就能把温度降下来。

租车的两天太阳很大，很晒很热，但天窗玻璃感觉并不热，只是手摸上去很烫，除此之外并没有什么感觉。

噪音方面，感觉和自己的油车噪声水平区别并不是很大，噪声源主要是胎噪和风噪，和油车的发动机噪声不太一样，没有什么不舒服的感觉。

电车停着可以开空调真是香。

从Ubuntu迁移到Debian：使用NetworkManager管理网络连接

发表于 2022-07-05 更新于 2023-04-24 分类于 记录 ， Debian

在物理机上安装Debian之前，我先在虚拟机上体验了一段不短的时间，发现了Ubuntu和Debian的一个小区别：Ubuntu安装好之后通过gnome系统的设置功能是可以配置网络的，而Debian安装好之后在gnome系统设置中都看不到网络连接。后来各方查找，发现需要对NetworkManager进行一些配置才能让其正确接管网络配置。

系统/etc/NetworkManager下的NetworkManager.conf配置文件中有一个[ifupdown]字段，其中有一个managed配置项，取值true时由NetworkManager管理网络连接，此时在gnome系统设置中就能看到网络连接了。

另外我在设置网卡固定IP时发现，通过gnome系统设置网络后，需要重新启动设置才能生效，不知道是估计设计成这样的还是BUG，虽然有点麻烦但还不太影响使用。

还有后来我在物理机上安装好Debian之后，发现NetworkManager的这个设置选项即使是false，在gnome系统设置中也是能看到后接入的USB有线网卡连接的…

搞得我都有些迷糊了…

从Ubuntu迁移到Debian：选择带无线网卡支持的netinst镜像

发表于 2022-07-05 更新于 2023-04-24 分类于 记录 ， Debian

在2022年当下，相当数量的笔记本都已经不提供有线网络接口了，如果想以netinst镜像方式安装Debian就只能通过无线网卡联网（有USB网卡的除外），而Debian的官方netinst镜像不带non-free的firmware，很多机器上启动之后因为无线网卡不能使用而无法继续安装过程，因此需要改用unofficial的附带non-free版本firmware的镜像以便使用无线网卡。

unofficial的镜像在官方以及各镜像站都有提供下载，路径一般是类似debian-cdimage/unofficial/non-free/cd-including-firmware这种。例如我一般使用合肥中科大的镜像，就是http://mirrors.ustc.edu.cn/debian-cdimage/unofficial/non-free/cd-including-firmware/11.3.0+nonfree/amd64/iso-cd/firmware-11.3.0-amd64-netinst.iso

另外unofficial下可能在images-including-firmware这个路径下也能找到这个带firmware的镜像，通过checksum对比可以知道两个路径下的镜像文件是完全相同的，使用任意一个都行。

从firmware版本镜像启动后，可以通过ip link命令查看无线网卡是否正常识别。
