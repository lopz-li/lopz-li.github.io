使用Wine运行VirtualNES模拟器

发表于 2021-12-16 更新于 2023-04-24 分类于 记录 ， 模拟器 ， Wine ， NES

任何时候都不能忘记了游戏。

在Ubuntu下折腾了几天之后还是想要玩一下游戏，于是找了各种原生NES模拟器来折腾，最后发现效果都不尽人意，几乎色彩都不自然，而且模拟效果也不好，各种卡顿延迟。

有了之前用Wine运行WinX YouTube Downloader的经验，决定也来跑一下Windows下的模拟器试试。直接把VirtualNES拷过来，在终端执行，还是报GLX错误。加上环境变量再跑，又报什么d3d内存不足啥的。网上搜了下，发现要在注册表中设置一下VideoMemorySize，像下面这样，512就够用了（键值没有的话手动创建）。

再次运行，成功！！！

这样每次运行都要打命令有点麻烦，于是准备写一个shell，每次双击就可以了，这下跳坑了…

直接写脚本，加上自动补全绝对路径之后，wine总是报各种找不到exe文件，完全摸不着头脑。

经过一白天的研究，总结了以下几条

  1. 需要设置WINEPREFIX环境变量，指向wine的配置环境，一般都是/home/username/.wine这样（就是用户主目录下的.wine，winecfg配置好的环境一般都在这里）
  2. 没有任何指定的情况下，wine默认会在配置环境目录下去找exe文件，所以要么cd到exe文件所在路径后再执行wine，要么在wine参数中带exe文件的完整路径

确定了原则之后，重新写了脚本，执行一次成功。

于是复制了一份WinX YouTube Downloader的desktop快捷方式文件，稍微改一下作为VirtualNES的启动图标，在~/.local/share/applications下创建一个符号链接即可在应用列表里看到它。

完工！！！

Ubuntu环境中使用YouTube下载器

发表于 2021-12-15 更新于 2023-04-24 分类于 记录 ， Wine

今天尝试了Ubuntu环境下的好几个YouTube下载器，诸如you-get/youtube-dl之类的，不是速度慢就是完全下不动，和Windows下的WinX YouTube Downloader完全没法比，看来还是离不开它啊。

于是想到使用wine来运行WinX YouTube Downloader。安装过程非常顺利，装好后也能看到软件图标，但是启动之后会闪退。在命令行中直接用wine启动exe程序发现提示和GLX有关的错误。

上网搜索发现有一篇文章提到需要设置环境变量

照做之后再次启动程序成功，试着下载了一个视频，速度飞快。

但是把环境变量放到用户配置文件之后，重新进系统的时候发现通过图标启动应用还是闪退，并且应用列表界面出现颤动，也无法相应鼠标滚轮了。于是尝试修改应用desktop文件，将环境变量加到命令行上，又发现desktop文件中的命令行本身有问题，索性将其修改为之前在命令行启动成功时使用的命令格式，执行成功。重启系统后通过图标启动应用成功，再次尝试下载文件，一切正常。

折腾的开始

发表于 2021-12-11 更新于 2023-04-24 分类于 其他

缘起

多年前研究C语言编译的时候在MSN的LiveSpace写过几篇笔记，后来太懒就没有继续了。没过几年连LiveSpace自己都挂掉了，那几篇笔记都找不回来了，想想还真是有点唏嘘。最近突然又想没事记点东西了，就在gitee上捡起来吧。

一些想法

大学起就一直在折腾Linux，后来工作中也用得不少，算起来有好些年了。但是仔细想来，因为游戏、专业软件等各种原因，一直都还是离不开Windows。不知不觉地，最近发现几乎工作生活娱乐中使用电脑的需求Linux环境基本都可以满足了，那借着这回折腾blog的契机，不如尝试一下所有事情都在Linux环境下完成看看。

准备工作

首先当然要装上系统并且折腾好hexo环境等等，写这些字的时候基本环境已经弄得差不多了，还是费了几天功夫的。

期待后续

写了这些字作为开头，也是一个发布效果测试，希望这次能坚持得久一点，不管是写东西还是折腾工作环境这个事。期待发布下一篇…
